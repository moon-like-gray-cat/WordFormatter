# WordFormatter —— Word 文档编号与格式一键规范化工具

**WordFormatter** 是一款用于 **自动整理 Word 文档标题编号与段落格式** 的轻量级工具。  
它能帮助用户快速清洗文档结构、统一标题样式，并将手动编号转换为标准格式，从而方便生成目录、规范论文或课程报告的排版。

## ✨ 功能简介

- **统一标题格式**  
  通过用户在选择的标题样式识别各级标题编号（如：`一、`、`(一)`、`1.` 等），并自动应用 Word 自带的“标题1/标题2/标题3……”样式。

- 针对`(`的中英文格式，会被统一中文格式下的`（`
- **规范字体样式**  
  - 标题自动统一字体、字号和加粗规则  
  - 正文格式自动统一  
  - 图题、表题自动规范格式、统一字号并居中

- **一键生成目录**  
  经处理后的文档可直接在 Word 中通过：  
  **引用 → 目录** 生成目录。

## ⚠ 使用前请注意

### 0.固定行距问题
如果正文的行距是固定的磅值，会压缩图片的显示空间。简单来说就是图片显示不全
非必要不要设置固定行距，不然只能自己手动调整行距的设置了（先全选中设置为一倍行距，这个时候图片会恢复正常，之后再单独选中文字，设置行间距）
### 1. 文件格式必须是 `.docx`
`.doc` 无法处理，请先另存或转换。

---

### 2. 编号规则（非常重要）

Word 中有内置自动编号系统，其特点是：

- 编号无法单独选中  
- 删除/插入内容时编号会自动变化  

👉 **本软件暂不支持识别Word 内置的自动编号。  
仅支持“手动输入”的编号（纯文本）。**

---

### 3. 段落必须使用“硬回车（↵）”
请确保段落都是按 **Enter** 换行的硬回车（↵），而不是 Shift+Enter 的软回车（↓）。

---
### 4. 编号必须单独成段

因为程序通过“段落前缀识别编号”，所以标题编号不能与正文混在一行。
如果（1）是您规定的标题之一

正常示例：
```angular2html
(1)标题
text
(2)标题
text
```

错误示范
```angular2html
(1)text(2)text
```
只会识别（1），text(2)text 都会当作(1)的标题

### 5. 注意正文开头不要出现可能被识别的编号

如果你的编号使用了：  
`一`、`1`、`a` 等常见字符，

请避免正文段落以这些字符开头，否则可能被判断为标题编号。
